2022-11-08

Introduction

This case study uses the Ames Housing dataset to try to predict a propertys sales price using other factors. The objective for this study is to enable Housing Investment Gurus (HIG) to make decisions on where to invest; properties with factors indicating they will sell for higher will make better investments. Neighborhood Value

One of the tasks assigned in the case was to select around three data proxies to be used to determine a neighborhoods value for HIG. As HIG is a real-estate investment firm running a business, it would be within their interest to practice business where sales are happening frequently and profitably. Therefore some data proxies to track high value neighborhoods would include metrics such as number of sales made and median sale price in the last month. (Number of listings would also be considered, but in this database we only have listings for sold houses.)

We can also get a high-level look at what features may be associated with higher prices through correlation between sale price and numeric variables. Ordinal variables may also be considered, so those must be converted to numeric.

Modeling and Diagnostics

Our initial model will be a linear model that uses every variable to predict SalePrice. This model does not use all 80 variables, we used nearZeroVar to reduce the number of columns considered.

dim(train)
## [1] 1457   60
#linear model
train.md <- subset(train, select=-c(Id))
train.md <- train.md[, names(which(colSums(is.na(train.md)) == 0))]
my.cols = names(train.md)
names(train.md) <- make.names(names(train.md))
my.lm <- lm(log(SalePrice) ~., data=train.md)
lm.rmse = sqrt(mean((my.lm$residuals)^2))
linear.error = data.frame(RMSE = lm.rmse, Rsquared = summary(my.lm)$r.squared)

par(mfrow=c(1,2))
plot(my.lm, which=c(1,2))

We check if the linear model adheres to linear regression assumptions of linearity, constant variance, independence, and normality with the residual and Q-Q plots. The residuals appear normally distributed, centered around zero, and have no relationship to the fitted values. The Q-Q plot shows that the distribution has more extreme values than would be seen from a true normal distribution, but the points largely fall on a straight line so can be considered acceptably normal.

The RMSE and R-squared values of this linear model are below:

linear.error
##        RMSE  Rsquared
## 1 0.1253243 0.9028047

Our RMSE is quite low and our R-squared is high on average, this linear models predictions were close to the actual values, and about 90% of variability in the response data (SalesPrice) is explained by the model. Based on these metrics, the linear model is performing well.

However, this is likely a case of overfitting on the training data, since we are using so many explanatory variables to predict SalesPrice. The objective of the study is to predict future sale prices with unseen data. One way to solve this problem is through regularization. We first try fitting a ridge regression model to the data, using 10-fold cross validation and lambda values ranging from \(10^{2}\) to \(10^{-3}\):

#something that isnt just a linear model
y = log(train.md$SalePrice)
x <- subset(train.md, select=-c(SalePrice)) %>% data.matrix
#ridge
lam.list = 10^seq(2, -3, by = -.1)
cv_ridge <- cv.glmnet(x, y, alpha=0, lambda=lam.list)
#cv_ridge$lambda.min
plot_glmnet(cv_ridge$glmnet.fit)

We can loosely interpret some of the relationships between variables using the GLM ridge regression plot. For example, MSZoning is one of the first variables to enter the model (that is, its coefficient value leaves 0) and stays as a variable with a high coefficient value. We see that ExteriorQuals coefficient value increases, then decreases as OverallQual increases, which we can attribute to the two variables having colinearity, as a property with high exterior quality would have high overall quality.

The error metrics of the ridge regression are below.

ridge.trainerror <- predict(cv_ridge, newx=x, s = "lambda.min") %>% caret::postResample(., y)
ridge.trainerror
##      RMSE  Rsquared       MAE 
## 0.1658390 0.8345860 0.1031762

The ridge regressions RMSE and R-squared are worse than the linear model, but this is to be expected since the idea behind ridge regression is to penalize overfitting coefficients, so its training error may be worse than the linear model.

We can also try fitting a lasso regression. Since coefficient values can be reduced to 0 in lasso regression, it can give us a better idea of which features may be more significant while also avoiding overfitting like ridge regression by yielding simpler models with fewer coefficients.

#lasso
out = glmnet(x, y)
cvfit = cv.glmnet(x, y, alpha=1,nfolds=10)
#coef(cvfit, s="lambda.min")
plot_glmnet(cvfit$glmnet.fit)

The order of magnitude of coefficients remain the same from ridge regression, although the values and paths taken are different, and the total number of coefficients remaining at the end is smaller in the lasso regression. We can also see the order of variables entering the model more clearly.

lasso.trainerror <- predict(cvfit, newx=x, s = "lambda.min") %>% caret::postResample(., y)
lasso.trainerror
##      RMSE  Rsquared       MAE 
## 0.1780432 0.8154295 0.1186117

The training error on the lasso regression is slightly worse than the ridge regression, but this is expected behavior as the lasso is a simpler model.

Choosing one of the models becomes a matter of deciding between the risk of overfitting or underfitting (the bias-variance tradeoff). In this particular case, we want to be able to apply the model to future data and in other regions, so we choose to use the lasso regression despite its higher training error.

Conclusions and Future Work

Using lasso regularized regression, we were able to create a model with RMSE of approximately \(0.178\) and R-squared of \(0.815\) to predict the sales prices of properties from the test data. We can rank feature importance by the order of their entry into the regression model during lasso regularization. Unfortunately post-selection inference goes beyond the scope of this case and so more rigorous tests of coefficients statistical significance will yet to be explored.

Likely the greatest pitfall of the model is that it fails to acknowledge the three most important elements of real estate, those being location, location, and location. This is especially egregious considering that we want to be able to apply the model to other locations. Since the extent of locational information in the given dataset was mostly limited to nominal variables about the propertys zoning classification or not varied enough to contribute to the model, it may be helpful to have quantitative information such as a propertys latitude, longitude, and elevation, which are easier variables to mathematically model. Even just with the data given, the lasso regression may be underfit, considering that its error was worst among the three models considered, and more complex modeling techniques such as elastic net regression or non-linear regression could have been better picks. Finally, there may be some ethical considerations given that one of the highest-contributing features of the model is Overall Quality, which is a heuristic measurement by local realtors who are not immune to bias; racial discrimination in housing appraisals is a well-documented phenomenon.
